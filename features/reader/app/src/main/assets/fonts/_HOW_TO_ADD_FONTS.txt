HOW TO ADD A FONT TO KANSHU
===========================

Drop your font files in THIS directory, then add one enum entry. The file is
purposely named with a leading underscore so it sorts above the font files
when you `ls` and is easy to spot.

Files to drop here
------------------
Prefer variable fonts with a wght axis — one regular file covers the whole
boldness slider. Naming convention, matching the existing files:

    <Family>-VariableFont_wght.ttf      (variable; axes may vary, e.g. opsz,wght)
    <Family>-Regular.otf / -Bold.otf    (static fallback pair)
    <Family>-OFL.txt                    (or whichever license the font ships with)

LICENSE FILES ARE REQUIRED. Keep the upstream license text in this folder —
don't strip it.

One-step wire-up
----------------
:core:model — add an enum entry
   File:  core/model/src/main/kotlin/com/charliesbot/kanshu/core/reader/ReaderFont.kt
   Add:   AtkinsonHyperlegible("Atkinson Hyperlegible", "fonts/AtkinsonHyperlegible-VariableFont_wght.ttf"),

The reader engine loads `regularAssetPath` directly via Typeface.Builder and
drives boldness along the font's wght axis. Two per-font knobs:

- `maxFontWeight`: set it to the upper end of the font's wght axis (from its
  fvar table) when it tops out below 900, so no boldness slider step is a
  dead tap. LibreBaskerville (700) is the existing example.
- `boldAssetPath`: only for static fonts with no wght axis — they flip to the
  bold file when boldness is on. OpenDyslexic is the existing example.

Display name
------------
The "Aa" chip label in the Font tab (FontTab.kt) comes from the enum's
`displayName`. Keep it human-readable ("Atkinson Hyperlegible"), not the file
stem ("AtkinsonHyperlegible").
